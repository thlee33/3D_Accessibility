# 강남역 3D 복셀 접근성 지도

건물의 접근성을 하나의 점이나 하나의 평균값으로 표현하면, 같은 건물 안에서 실제로 달라지는 공간 경험을 놓치기 쉽습니다. 역과 가까운 1층 모서리, 뒤쪽 저층부, 상층부는 같은 주소를 공유하지만 도달하는 데 드는 시간과 체감은 다릅니다.

이 프로젝트는 강남역 주변 건물을 10m 단위의 3D 복셀로 나누고, 각 복셀의 접근시간을 색으로 보여주는 실험입니다. 목적은 “역세권 접근성”을 건물 단위가 아니라 건물 내부의 3차원 공간 단위로 읽어보는 것입니다.

## 왜 만들었나

일반적인 역세권 분석은 역 중심점이나 건물 중심점까지의 거리를 기준으로 합니다. 하지만 도심 고밀도 지역에서는 이 방식이 너무 거칠 수 있습니다.

- 같은 건물 안에서도 역과 가까운 면과 먼 면이 있습니다.
- 1층과 고층부는 수직 이동 비용이 다릅니다.
- 대형 건물은 하나의 평균값으로 표현하기 어렵습니다.
- 오피스나 상점의 실제 가치는 호실 위치, 층, 외부 노출, 조망, 접근성의 조합으로 결정됩니다.

그래서 이 PoC는 건물을 잘게 쪼개 각 복셀마다 접근시간을 계산합니다. 지도 위의 색은 “이 건물에 접근하기 쉬운가?”가 아니라 “이 3D 공간 셀에 도달하기 쉬운가?”를 의미합니다.

## 접근시간은 어떻게 계산했나

접근시간은 수평 이동거리와 수직 이동거리를 더한 뒤, 보행속도 4km/h로 나누어 계산했습니다.

```text
access_time_min = (horizontal_distance_m + vertical_distance_m) / 66.67
```

여기서 66.67m/min은 4km/h를 분 단위로 환산한 값입니다.

다만 층별 계산 방식은 1층과 2층 이상을 다르게 처리했습니다.

### 1층

1층 복셀은 각 복셀 중심에서 가장 가까운 강남역 출입구까지의 유클리드 수평거리를 직접 계산합니다.

```text
1층 접근거리 = distance(voxel_xy, nearest_exit_xy) + z
```

따라서 같은 건물 1층 안에서도 출입구와 가까운 쪽과 먼 쪽의 접근시간이 다르게 나타납니다.

### 2층 이상

2층 이상은 엘리베이터나 계단 위치를 모르면 층 내부 수평 이동을 알 수 없습니다. 그래서 건물별 대표 수평거리 하나를 사용합니다.

```text
2층 이상 접근거리 = building_representative_horizontal_distance + z
```

건물 대표 수평거리는 해당 건물의 1층 격자 중 지하철 출입구와 가장 가까운 셀의 거리입니다. 이 방식은 알 수 없는 실내 동선을 과도하게 추정하지 않고, 상층부에서는 수직 이동에 따른 차이만 표현합니다.

## 색상은 무엇을 의미하나

복셀 색상은 접근시간 등급입니다.

| 색상 등급 | 접근시간 | 의미 |
|---|---:|---|
| 파랑 | 0-3분 | 매우 가까움 |
| 청록 | 3-5분 | 가까움 |
| 노랑 | 5-7분 | 보통 |
| 주황 | 7-10분 | 멂 |
| 빨강 | 10분 초과 | 매우 멂 |

이 색상은 건물 전체의 평균 접근성이 아니라 각 복셀의 접근시간을 나타냅니다.

## 현재 데이터

- 대상지: 강남역 중심 반경 650m
- 건물 데이터: OpenStreetMap 기반
- 표시 건물: 447개
- 지하철 출입구: 5개
- 복셀 크기: 10m x 10m x 10m
- 복셀 수: 13,361개
- 최소 접근시간: 약 0.12분
- 최대 접근시간: 약 11.04분

## 코드화 방식

데이터 생성은 크게 네 단계로 구성됩니다.

1. OSM 건물과 출입구 데이터를 수집합니다.
2. 좌표를 EPSG:5179로 변환해 미터 단위 거리 계산이 가능하게 만듭니다.
3. 반경 650m 안에 10m 격자를 만들고, 건물 footprint와 겹치는 셀만 남깁니다.
4. 건물 높이에 따라 Z 방향 복셀을 만들고, 접근시간을 계산해 `voxels.json`으로 저장합니다.

웹에서는 이미 생성된 정적 JSON만 읽습니다. CesiumJS에서 건물은 반투명 3D polygon으로, 접근성은 BoxGeometry 복셀로 렌더링합니다. 그래서 이 결과물은 서버 계산 없이 정적 웹페이지로도 볼 수 있습니다.

## 이 방식의 한계

이 PoC는 접근성을 이해하기 위한 첫 모델입니다. 다음 한계가 있습니다.

- 실제 보행 네트워크 최단거리 대신 출입구와의 유클리드 거리를 사용합니다.
- 지하상가, 실내 연결통로, 횡단보도 대기, 경사로 등은 반영하지 않았습니다.
- 건물 내부 엘리베이터, 계단, 코어 위치를 알 수 없어 2층 이상은 건물 대표 수평거리로 단순화했습니다.
- OSM 건물 높이 정보가 없는 경우 추정 높이를 사용합니다.

그럼에도 이 모델은 “건물 내부 3D 공간 단위로 접근성을 계산한다”는 관점을 실험하기에는 충분합니다.

## 접근성을 넘어 3D 공간가치 평가로

접근시간은 공간가치를 설명하는 여러 요소 중 하나입니다. 같은 복셀 구조에 다른 지표를 붙이면, 더 넓은 3D 공간가치 평가로 확장할 수 있습니다.

- 전망/조망: 한강, 공원, 도로, 랜드마크가 얼마나 보이는가
- 가시성: 주요 보행축이나 도로에서 해당 공간이 얼마나 잘 보이는가
- 일조/그늘: 시간대별 빛과 그림자 조건은 어떤가
- 소음/환경: 도로, 철도, 혼잡 지역과 얼마나 가까운가
- 시장성: 층별 임대료, 공실률, 매출 추정치와 어떤 관계가 있는가

이 요소들을 복셀별로 결합하면 건물 전체가 아니라 오피스나 상점의 호실 단위에 가까운 3D 공간가치를 추정할 수 있습니다.

```text
space_value
  = 접근성 점수
  + 전망/조망 점수
  + 가시성 점수
  + 환경 품질 점수
  + 시장 데이터 보정
```

즉, 이 지도는 단순한 접근성 시각화가 아니라 도시 공간을 3D 셀 단위로 평가하는 더 큰 분석의 출발점입니다.
